# Project Tehuti Style Guide

## Voice
- Direct, calm, authoritative. No hype.
- Evidence-first: show actions and outputs before conclusions.
- Avoid filler and vague claims. Always tie statements to observed evidence.

## Visual Language
- Palette: gold on obsidian. Use existing theme tokens.
- Panels: use gold borders and sand text for data.
- Titles: keep short and purposeful.

## Output Structure
- Prefer short, readable sections.
- For tool actions: show a short execution log with inline evidence when brief.
- Use the Evidence panel only for longer outputs.
- Avoid auto-summaries; let evidence and the final response carry the result.
- Avoid raw markdown in responses.

## Status Symbols
- `◆` start of a tool call
- `✓` completion of a tool call
- `✗` failed tool call
## Errors
- Show the error and next action.
- Never leave the user with a "No response" message.

## Commands & Help
- Command descriptions should be verbs and task-focused.
- Keep help tables aligned and consistent.
- Transcript view should be plain and evidence-first.

## Pickers
- Pickers should be search-first, fast, and minimal.

## Branding
- Use Tehuti mythos language sparingly and consistently.
- Avoid generic or unrelated slogans.
